Smile Classifier

A web app that decides whether a face is smiling or not smiling, built with classic machine learning.

What this app does

Upload a photo of a face and the model predicts one of two classes — Smiling or Not Smiling — along with a confidence score. Every classification is stored so you can review past results on the History page.

Framework & model — and why

The classifier is built with scikit-learn, a mature and lightweight machine-learning library. For this two-class image problem it is an excellent fit: it trains fast, needs no GPU, and is simple to deploy.

The model is a Support Vector Machine (SVM) trained on HOG (Histogram of Oriented Gradients) features. HOG describes the shape and edges of a face — the curve of the mouth, the raised cheeks of a smile — rather than raw pixel brightness, which makes it robust to lighting. An SVM then finds the boundary that best separates smiling from non-smiling faces. This pairing is accurate on modest datasets and far simpler than a deep neural network, which would be overkill here.

How the training script works

  • Every image is converted to grayscale and resized to 64×64 so all inputs are uniform.
  • HOG features are extracted from each image, turning it into a vector of numbers the model can learn from.
  • A StandardScaler + SVM pipeline is trained; 20% of the data is held back to measure honest accuracy on unseen images.
  • The trained pipeline is saved to a pickle file so it can be reused without retraining.

How the inference script works

  • It first checks that the trained pickle model exists; if not, it asks you to train first.
  • The uploaded image is preprocessed with the exact same steps as training (grayscale → resize → HOG), guaranteeing consistency.
  • The model predicts the class and a confidence score, which is shown to you and saved to the database.

Technology stack

  • FastAPI + Jinja2 — web framework and templating
  • scikit-learn + scikit-image — model and HOG features
  • Pillow — image processing and JPG conversion
  • PostgreSQL + SQLAlchemy — database and ORM
  • Docker — containerized, reproducible deployment
